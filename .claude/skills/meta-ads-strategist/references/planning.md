## 1. Стратегическая рамка 2026: Andromeda-эра

**Механика.** Andromeda — нейросетевой retrieval-движок Meta (анонс дек. 2024,
глобальный роллаут завершён ~окт. 2025): система **читает сам креатив** и по нему
решает, кто eligible его увидеть. Аудиторные настройки из жёстких ограничителей
превратились в мягкие подсказки. Оценка практиков: креатив объясняет ~56%
дисперсии перформанса (directional). Следствие для планирования: бюджет на
языковую кампанию без конвейера креативов на этом языке — мёртвый бюджет.
**Пропускная способность студии (melia-montage) на язык = реальный потолок
перформанса этого языка.**

**Доктрина консолидации.** Фрагментация теперь стоит денег дважды:
1. Пересекающиеся адсеты торгуются друг против друга в аукционе → CPM
   раздувается на ~10–20%.
2. Каждый адсет должен сам набрать ~50 оптимизационных событий/неделю для
   выхода из learning phase; дробление сигнала держит все адсеты в Learning
   Limited. Рабочий пример из ресёрча ровно нашего масштаба: 4 адсета по
   $50/день дают ~14 конверсий/нед каждый и не выходят из learning никогда;
   один консолидированный адсет на том же бюджете — ~56/нед и выход за 7 дней.

Целевая структура 2026: **1–3 адсета на кампанию, 6–20 действительно разных
креативов в каждом** вместо легаси 5–10 сегментированных адсетов по 3–5 адов.
Тест Scaledon: 1 адсет × 25 креативов побил 5-адсетную структуру на +17%
конверсий при −16% стоимости (single-source, directional).

**Что умерло и почему:**

| Практика | Почему мертва |
|---|---|
| Interest stacks («luxury real estate» + «investing») | Andromeda переопределяет интересы; таргетинг делает креатив. Опции detailed targeting планомерно выпиливаются |
| 1% lookalikes как основной prospecting | Поведенческий сигнал Meta богаче seed-аудитории; LAL сужает delivery, не улучшая качество. Выживают только как «suggestions» в Advantage+ Audience |
| Микросегментация (адсет на интерес/возраст/LAL-тир) | Дробит сигнал ниже порога learning + аукционный оверлап; вдвойне фатально при 500–3,000 EUR/мес/рынок |
| «CBO всегда выигрывает» (доктрина 2019–2020) | На малых бюджетах CBO сливает всё в адсет, победивший в первые часы, и в самый дешёвый рынок |
| Ручной выбор плейсментов (feed-only) | Исключения плейсментов дисквалифицируют из Advantage+ state и сужают matching-пространство Andromeda |
| Один «победивший ад» на месяцы | Срок жизни креатива сжался с 6–8 до 2–4 недель; Andromeda вознаграждает портфель из 6+ разных концептов |
| Оптимизация на сырой объём форм | CPL растёт ~20% YoY, volume-формы всё чаще везут мусор; стандарт 2026 — Conversion Leads goal с CRM-фидбеком через CAPI |

Для Meliá это означает: рычаги, которые у нас остались, — **(а) разнообразие
креативов на язык, (б) качество сигнала (CAPI, Meta lead_id, EMQ), (в)
структурная простота, (г) терпение в learning phase**. Всё остальное — шум.

## 2. Структура аккаунта для Meliá

### 2.1 Кампания × язык, не × страна

Правило: **одна кампания на язык креатива** (RU, SR, EN, DE, PL, HE — согласно
языкам проекта), внутри — один broad-адсет, группирующий страны, которые этот
язык обслуживает. Язык важнее страны, потому что Andromeda матчит креатив с
пользователем: сербский ад сам себя таргетирует. Нативная локализация (не
авто-перевод — см. правило транскреации в project-context.md) удваивает CTR
по нескольким источникам.

Два ограничителя на группировку гео внутри адсета:
- **Не смешивать рынки с CPM-разбросом >30% (или >3x по стоимости)** под одним
  бюджетом — алгоритм зальёт бюджет в дешёвый аукцион (Балканы), потому что это
  «экономически рационально» для него и катастрофично для нас.
- CPM по нашим рынкам различаются в 2–4 раза: Сербия ~$2.8, Польша ~$5.5,
  Израиль ~$7.5, Германия ~$10, UK ~$10.3 (Черногория в датасетах отсутствует,
  ожидать уровень Сербии). Детали — [benchmarks.md](benchmarks.md).

Практическая раскладка: RU-кампания при необходимости делится максимум на два
адсета по CPM-тирам (Балканы/дешёвые гео vs DE/IL-тир — из ресёрча: RU-диаспора
по странам проживания DE/RS/ME/CY/IL), и второй тир добавляется **только** когда
бюджет тянет оба (см. learning math ниже). SR-адсет — Сербия+Черногория+Босния
(похожие CPM — можно в один). Не строить матрицу «кампания × язык × страна»:
при 500–3,000 EUR/мес/рынок она гарантирует вечный Learning Limited везде.

### 2.2 ABO vs CBO при наших бюджетах

При 16–100 EUR/день на рынок мы **однозначно в зоне ABO / single-ad-set
кампаний**. Механика: CBO перераспределяет бюджет к «победителю первых часов» и
систематически перекармливает самый дешёвый рынок; ABO гарантирует каждому
концепту/гео его ~50 событий. CBO становится уместен только если языковая
кампания позже консолидирует 3+ проверенных адсета, каждый из которых стабильно
набирает ~50 лидов/нед — до этого далеко.

В API это уже зашито в money-safety (project-context.md):
`is_adset_budget_sharing_enabled=false`, `daily_budget` на уровне адсета,
`bid_strategy=LOWEST_COST_WITHOUT_CAP`.

**Bid strategy:** стартовать на Highest Volume. Cost Per Result Goal — только
после 2–4 недель стабильных данных и с таргетом **на уровне или выше фактически
достигнутого CPL**: для lead-объективов кап «дырявый» (факт обычно выше
таргета), а заниженный таргет обрушивает delivery. Для high-ticket CPL-кап —
вообще не тот рычаг: качество контролируем CRM-сигналом (см. 2.5), а не капом.

### 2.3 Сколько чего

| Уровень | Норма | Механика |
|---|---|---|
| Кампаний | 1 на активный язык; активных языков — сколько тянет бюджет (см. §3.2), не все 6 сразу | Каждая кампания = кластер креативов одного языка |
| Адсетов в кампании | 1 (второй — только retargeting или второй CPM-тир в самом финансируемом рынке) | Каждый адсет должен сам выходить на ~50 событий/нед |
| Креативов в адсете | 6–10 живых, действительно разных (лимит платформы 50/адсет — не ориентир) | Разные хуки/персоны/стадии осознанности, микс 9:16 видео + 4:5/1:1 статика; вариации одного концепта на разных языках = один концепт |
| Refresh | Модель 50/30/20 по креативным слотам: проверенные / итерации победителей / новые ставки | По сигналу (CPL растёт, CTR падает), не по календарю. Углы и фатиг — [creative.md](creative.md) |

### 2.4 Learning phase math — жёсткая арифметика структуры

Формула: **мин. дневной бюджет адсета ≈ ожидаемый CPL × 50 / 7**
(50 оптимизационных событий на адсет за 7 дней — порог выхода из learning).

На наших ожидаемых instant-form CPL (тиры из [benchmarks.md](benchmarks.md)):

| Рынок (тир) | Ожидаемый CPL | Мин. €/день | Мин. €/мес | Вердикт при 500–3,000 EUR/мес |
|---|---|---|---|---|
| Балканы / RU-диаспора в дешёвых гео | €8 | ~€57 | ~€1,700 | Достижимо при верхней половине бюджета |
| То же, оптимистично | €10 | ~€71 | ~€2,100 | Достижимо только у топ-рынка |
| PL | €10–25 | €71–179 | €2,100–5,400 | На грани / нет |
| DE / UK / IL | €25–60 | €179–429 | €5,400–12,900 | Недостижимо — Learning Limited терпим осознанно |

Выводы, которые нельзя обходить:
- Большинство рынков тянут **ровно один адсет**. Второй (retargeting) —
  только в лучше всего финансируемом рынке.
- 50/нед — ориентир, не шлагбаум: лид-кампании на дешёвом событии instant form
  работают и ниже порога, просто с большей дисперсией. Именно поэтому
  оптимизируемся на instant form lead (дешёвое событие в платформе), а не на
  сайтовые/глубокие CRM-события — те на наших бюджетах не выйдут из learning
  никогда.
- **Learning resets:** изменение бюджета >~20%, смена таргетинга/бида/
  события оптимизации, пауза 7+ дней; добавление/удаление адов — официально
  тоже (на практике сброс от добавления обычно мягче — tracking.md).
  Программные корректировки бюджета через fb CLI/MCP — шагами <20%.
- Learning Limited 2+ недели подряд = объективный сигнал «рынок недофинансирован»
  → правило паузы из §3.2.

### 2.5 Advantage+ leads (v25): что включать сейчас, что потом

В Marketing API v25 флага `smart_promotion_type` больше нет: Advantage+ leads —
это не тип кампании, а **состояние**, в которое кампания попадает при трёх
рычагах одновременно: (1) бюджет на уровне кампании с
LOWEST_COST_WITHOUT_CAP / COST_CAP / bid cap / min-ROAS, (2) аудитория geo-only
или `targeting_automation.advantage_audience=1`, (3) ноль исключений
плейсментов. Читается через `advantage_state_info = ADVANTAGE_PLUS_LEADS`.
Жёсткими контролями остаются только location, мин. возраст, exclusions, язык.

Наша позиция (и почему):
- **Сейчас включаем компоненты — бесплатные выигрыши на малом объёме:**
  `advantage_audience=1` (CRM custom audiences и язык — как seeds/suggestions;
  гео + мин. возраст + язык — жёстко), автоплейсменты. В малых не-US рынках
  seeds всё ещё полезны: диаспора — тонкий срез населения страны, хотя язык
  креатива при Andromeda фильтрует и сам.
- **Полный Advantage+ state — не сейчас, осознанно.** Рычаг (1) требует
  бюджета на уровне кампании, а мы на ABO (§2.2) — то есть текущая money-safety
  конфигурация *намеренно* не квалифицируется. Это соответствует практике 2026:
  доля полного автопилота у зрелых операторов **упала** (~38% → ~20% трекаемого
  retail-спенда к Q1 2026), потому что ему нужен большой объём событий.
  Meta-клеймы (~10% ниже cost per qualified lead, ~14% ниже CPL) предполагают
  достаточный сигнал.
- С ~фев 2026 новые leads-кампании стартуют со **всеми Advantage+ Creative
  enhancements включёнными по умолчанию** — аудировать per-ad: авто-модификации
  калечат локализованные RU/SR/HE оверлеи. Чек-лист — [launch.md](launch.md).

**Эндгейм — Conversion Leads performance goal** (оптимизация на качественный
лид по CRM-сигналу): клеймы Meta ~15–19% ниже cost per quality lead, +44%
lead→quality-lead. Жёсткие гейты: CAPI-синк CRM-стадий минимум ежедневный,
**~200 лидов/мес**, оптимизируемая стадия наступает в пределах 28 дней от лида,
её конверсия 1–40%, Meta lead_id хранится и возвращается (у нас инвариант уже
есть — см. CAPI-таксономию в project-context.md; стадия = `lifecycle_qualified`,
не сделка). Гейт 200/мес — ещё один аргумент за консолидацию: объём набирается
на весь аккаунт, а не размазывается. До достижения гейта — стандартная
оптимизация на лиды + квалифицирующие вопросы в форме ([launch.md](launch.md)).

## 3. Медиаплан-математика

### 3.1 CPL ceiling из юнит-экономики

Отраслевая рамка: маркетинг девелопера = 1–2% GDV (якорь 2%), из них paid —
30–60%. Формула потолка: **допустимый CPL = (допустимый paid-маркетинг на
сделку) × (конверсия лид→сделка)**.

Расчёт на наш вход €263,200 (по формуле дайджеста; в ресёрче worked example на
€200k — €3,000 маркетинга на сделку, ~€2,000 paid):

| Параметр | Значение |
|---|---|
| Маркетинг на сделку (1.5% от чека) | ~€3,950 |
| Из них paid media (~2/3) | ~€2,600 |
| CPL ceiling при лид→сделка 1% | ~€26 |
| CPL ceiling при 2% | ~€53 |
| CPQL ceiling (при ~30% квалификации) | CPL ceiling / 0.30 ≈ €87–175 |

Реалистичные конверсии для холодного международного off-plan (бенчмарки
2026): лид→сделка **1–2%** для планирования (paid social в недвижимости 1–4%;
сырые интернет-лиды 0.5–1.5%). Цепочка: лид → qualified 25–35% → встреча
30–50% от qualified → сделка 10–20% от встреч ≈ композитно ~1–2%. Дисциплина
CRM удваивает композит (структурные команды закрывают 5–10% vs 1.5–3% соло) —
то есть **удваивает допустимый CPL**; speed-to-lead <5 мин повышает вероятность
контакта до 21x — это рычаг сильнее любой медиа-оптимизации на нашем масштабе.

Потолки считать **по рынкам**, не по аккаунту: диаспорные RU/SR-лиды
квалифицируются и закрываются лучше холодных DE/UK → их ceiling выше при том же
чеке. Зашить потолки в CRM-дашборды как пороги pause/scale
([tracking.md](tracking.md)).

Санити-чек: полный маркетинговый cost-per-sale €2–6k на юнит €200k+ — внутри
нормы девелопера (1–3% цены). Наши сотни–низкие тысячи EUR/мес на рынок —
консервативно; **узкое место — конвейер креативов и обработка лидов, не спенд.**

### 3.2 Бюджет на рынок: полы и правило паузы

- **Пол активного рынка: ~€900–1,500/мес (€30–50/день).** Честный минимум
  тестирования — €1,500–3,000/мес; ниже ~€3,000/мес тесты только
  последовательные (одна переменная), не параллельные.
- При суммарных €1–2k/мес — **один** рынок; €4k — 2–3; только при €8k — 4–5.
- **Правило паузы:** если месячный бюджет рынка не покупает ~10–15 лидов по его
  ожидаемому CPL (примерно <€300–500/мес в дешёвых рынках, <€1,000–1,500 в
  дорогих) ИЛИ адсет сидит в Learning Limited 2+ недели — рынок ставится на
  паузу целиком, бюджет перераспределяется. Не держать €100–200/мес «для
  присутствия». Документированный кейс консолидации: 20 адсетов → 5 дали +29%
  ROAS и −25% затрат за 30 дней.
- Норма концентрации: **60–80% бюджета в топ-рынках**.

### 3.3 Какие рынки первыми и почему

Последовательность по формуле «стоимость сигнала × аффинити», фазовый вход
(Phase 1: минимально жизнеспособный тест $1–3k за 2–4 нед в одном рынке;
Phase 2: валидация $3–10k за 4–8 нед; Phase 3: масштаб):

1. **RU-диаспора + SR/Балканы** — самый дешёвый CPM в нашем наборе, высшее
   диаспорное аффинити (RU доминирует в сегментах проекта), нативный креатив
   уже в конвейере студии. Задача: накопить CAPI-сигнал и провалидировать
   воронку. Паттерн подтверждён кейсами branded residences: у Marriott
   Residences диаспорный таргетинг дал 110 из 150 qualified-лидов.
2. **PL** — дешёвый CPM (~$5.5), но аффинити не доказано: тестовая ставка.
3. **DE / EN-EU / HE** — последними и только с провалидированными креативами и
   отлаженной квалификацией: их CPL в 2–4 раза выше и сжигает малый бюджет
   быстрее всего.

Триггер расширения на следующий рынок: 4+ недели CPL под потолком **плюс**
первые qualified-когорты дозревают до встреч. Kill-триггер рынка: спенд
превысил 20–30x ожидаемого CPL при нуле qualified.

Аллокация — не по сырому CPL, а по **cost per qualified lead**: дешёвый CPM ≠
лучшая экономика; эффективная стоимость сделки = CPL рынка / лид→сделка рынка.

### 3.4 Сезонность (взвешивание годового бюджета)

- **+~30% в конце декабря–феврале**: новогодний всплеск (Rightmove — с Boxing
  Day) + пик поиска зарубежной недвижимости в конце февраля (Kyero, 91M
  визитов) + самые дешёвые CPM года (глобальный медианный CPM $15.74 в янв
  2026 vs $25.22 в ноя 2025).
- **+~20% в середине августа–сентябре**: второй пик Kyero (вернувшиеся
  отпускники); для Будвы — люди, только что видевшие Адриатику вживую:
  ретаргетить летних engagers.
- **−вес в ноябре**: пиковый CPM при слабом property-интенте — худший месяц.
- Базлайн март–июнь, мягкий ноябрь–начало декабря.

### 3.5 Forecast: когортная модель и её честные пределы

Метод: каждая месячная когорта спенда зреет через фиксированные конверсии и
лаговое распределение. Продажи месяца t = Σ по прошлым когортам
(спенд/CPL × композитная конверсия × вес дозревания). Веса дозревания сделок:
**5/15/25/25/15/10/5% на месяцы 2–8** от месяца лида.

Worked example из ресёрча (блендед €4,000/мес): RU/Балканы €2,000 @ €12 CPL ≈
165 лидов; PL €800 @ €20 = 40; DE/EN €1,200 @ €45 ≈ 27 → ~230 лидов/мес; при
30% квалификации ~70 QL/мес (CPQL ~€57 blended); при 1.5% лид→сделка ~3.5
сделки на когорту, размазанные на месяцы 2–8. **Первая сделка реалистично —
месяцы 3–5; steady state ~3–4 сделки/мес — только к ~7-му месяцу.** Донести
этот лаг до сейлзов заранее.

Честные пределы forecast:
- CPL-допущения — из US-тяжёлых бенчмарков и агентских диапазонов; наши
  фактические значения появятся только из своих когорт. Пересчитывать
  конверсии **ежемесячно из фактических CRM-когорт** — это же и петля CPQL для
  Conversion Leads.
- Цикл международного покупателя 5–7+ месяцев, luxury-нуртуринг ~20–25 касаний;
  Meta-лиды зреют дольше. Мерить когорты на горизонте 6–12 месяцев; месячный
  forecast первых месяцев — это лиды и qualified, не сделки.
- На сезонность умножать **строку лидов** (×1.3 янв–фев, ×1.2 конец авг–сен,
  ×0.8 ноя), не строку сделок.

## 4. Шаблон медиаплана (месяц/квартал)

Таблица месяца (пример на блендед €4,000/мес, цифры worked example §3.5;
конверсии: qualified ~30% лидов, встречи 30–50% от qualified):

| Рынок (кампания × язык) | Статус (70/20/10) | €/мес | CPL ож. | Лиды | Qualified (~30%) | Встречи (30–50% QL) |
|---|---|---|---|---|---|---|
| RU (диаспора, дешёвые гео) | proven — 70% | 2,000 | €12 | ~165 | ~50 | 15–25 |
| PL | growth — 20% | 800 | €20 | ~40 | ~12 | 4–6 |
| EN/DE (дорогой тир) | testing — 10% | 1,200 | €45 | ~27 | ~8 | 2–4 |
| **Итого** | | **4,000** | ~€17 blended | **~230** | **~70** (CPQL ~€57) | **21–35** |

Как пользоваться шаблоном:
1. Стройка плана: строки = активные рынки (не больше, чем позволяют полы §3.2),
   роли по портфелю **70/20/10** (proven/growth/testing) на базе
   cost-per-qualified, не CPL.
2. Рядом — 12-месячный когортный лист: строки = месячные когорты спенда по
   рынкам, столбцы = месяцы дозревания 1–8; лиды = спенд/CPL, qualified =
   ×~30%, сделки = ×1–1.5% с весами 5/15/25/25/15/10/5%.
3. Квартальный план = 3 месячные таблицы × сезонные множители §3.4 + вехи:
   гейт 200 лидов/мес для Conversion Leads, триггеры расширения рынков §3.3.

**Правило пересмотра (ежемесячно, в дату):**
- Пересчитать фактические CPL / % qualified / % встреч из CRM-когорт → обновить
  допущения forecast и потолки §3.1.
- Ребалансировка 70/20/10: рынок с CPQL хуже потолка 2 отчётных цикла подряд —
  вниз по портфелю или пауза (правило §3.2); triggers расширения — §3.3.
- Все программные изменения бюджетов через fb CLI/MCP — шагами **<20%**, иначе
  learning reset (§2.4). Money-safety процедура мутаций — project-context.md.
- Не принимать решений по рынку на 30-дневном окне: kill/scale-критерии внутри
  месяца — зона [tracking.md](tracking.md), судьба рынка — только по когортам.

## 5. Анти-паттерны планирования

- **Матрица «язык × страна»** как дефолт — множит Learning Limited адсеты;
  группировать одноязычные похожие по CPM страны, делить только при >30% CPM /
  >3x разницы стоимости.
- **Все 6 языков сразу** / trickle-funding «для присутствия» по €100–200/мес —
  норма 2026: 60–80% бюджета в топ-рынки, недофинансированные — пауза целиком.
- **Пул всех стран в один CBO** «пусть Meta оптимизирует» — при 2–4x разбросе
  CPM алгоритм зальёт всё в Балканы независимо от ценности лида.
- **Микросегментация аудиторий** (адсет на интерес/LAL-тир/возраст) и
  **lookalike-first планирование** — пост-Andromeda дробит сигнал и проигрывает
  broad + разнообразному креативу; LAL/интересы — только seeds.
- **Learning-порог как жёсткий гейт бюджета** (CPL × 50/7 для всех рынков) —
  для дорогих рынков это €4k+/мес на адсет; практика 2026: ориентир, терпимость
  к Learning Limited в дорогих рынках, здоровье меряем стабильностью CPL.
- **Оптимизация на сырой объём форм без CRM-фидбека** — везёт мусор при CPL
  +20% YoY; целевое состояние — Conversion Leads по `lifecycle_qualified`
  через CAPI.
- **Pixel-only трекинг** — ловит 60–70% событий и штрафует delivery под
  Andromeda; CAPI с дедупликацией — базлайн (наш инвариант Meta lead_id —
  project-context.md).
- **Частые правки и бюджетные скачки >20%** как «оптимизация» — каждый
  значимый edit сбрасывает learning; шаги <20%, refresh креатива по сигналу.
- **Прогноз «спенд этого месяца → продажи этого месяца»** — при пайплайне 5–7
  месяцев систематически врёт; только когортная модель дозревания.
- **Суждение о рынке по 30-дневному CPL/ROAS** — убивает лучшие prospecting
  кампании high-ticket; когорты 6–12 месяцев.
- **Вечный «победитель»** — срок жизни креатива 2–4 недели; планировать
  конвейер студии на язык, а не разовый запуск.
- **Дейпартинг/ручные плейсменты «для премиальной аудитории»** — достоверных
  бенчмарков поведения инвесторов по времени/плейсментам нет, а ограничения
  душат delivery; автоплейсменты + mobile-first 9:16/4:5, сэкономленные усилия
  — в удалённый «display suite» (видеозвонок + 3D-тур: у off-plan конверсия
  встреча→сделка 20–30%, с интерактивной визуализацией до ~50%).
- **Игнорирование Special Ad Category в плане** — housing-SAC точно действует
  для US/Canada, для «частей Европы» список мутный, а мультимодальный детектор
  2026 авто-флагает недвижимость по картинке и может молча урезать таргетинг
  API-кампаний. Перед масштабированием рынка — проверка эффективного таргетинга
  и SAC-статуса тестовой кампании; детали — [policy.md](policy.md).
